🤔 ¿Por qué SmarkForm es distinta?

  • 👌 Sencillez ( HTML + metadatos , Markup-agnostic , SoC ,
    Zero-Wiring (Contextual triggers / Directory-like paths)         )
  • 🚀 Poténcia ( JSON ⬆️ /⬇️ , Subformularios , listas dinámicas ordenables ,
    gestión de interdependéncia     , tipado y validación     )
  • 🫶 Usabilidad ( Navegación natural , Contextual hot-keys ,
    Plegado de secciones     , (Auto)ordenación...     )
  • ♿ Accesibilidad ( Tiene en cuenta aspectos de accesibilidad ,
    Aunque todavía queda trabajo por hacer...     , 🆘 Help!!     )
  • 🏗️ Extendible ( Posibilidad de incorporar nuevos tipos de campos ,
    Incluso fragmentos o "Mixin"s         )
  • 🚁 Independéncia ( Vanilla JS , ES module / Cjs ,
    CDN / NPM / GitHub         )
